# User Preferences

## Git

- Never push to the main branch without explicit permission
- Always use `/commit` for commits and `/pr` for pull requests
- Commit messages focus on why, not what. Let the code speak for itself

## Shell

- Use simple, single-purpose tool calls
- Don't use chained tool calls if it's going to require the user to babysit permissions
- Prefer `rg` to `grep` for fast searching

## Planning Center

- Working in the $HOME/Code folder is always Planning Center development
- Branches should use the prefix `danott/` or `proto/`
- `danott/` is used for identifying branches I "own"
- `proto/` is used for creating prototype environments when pushing to GitHub
- Ask if you're not confident which prefix to choose
